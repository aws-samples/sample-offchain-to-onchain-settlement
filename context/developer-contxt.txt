Design and implement a minimal end-to-end reference architecture for:

“Bridging Generic Financial Messages On-Chain using AWS + Chainlink CRE (Mint/Burn model)”

Goal: Generic off-chain financial messages (SWIFT/ISO20022/proprietary) MUST NOT execute on-chain. Instead:
- Ingest + store raw message off-chain
- Compute messageDigest = keccak256(raw message bytes)
- Normalize into Canonical Settlement Instruction (CSI)
- Deterministically derive instructionId (stable across retries)
- AWS signs CSI using EIP-712 typed data with an institutional key
- Chainlink CRE orchestrates submission + confirmation
- Smart contracts verify EIP-712 signature, enforce idempotency + expiry, and execute mint-only OR burn+mint

OUTPUTS REQUIRED

3) Chainlink CRE workflow (Web3 orchestration):

     - chainId (uint256)

   - Must be message-format agnostic: parsing only yields shared semantics (amount, parties, value time, reference)
